Components / Button

Button

Use buttons to trigger actions or navigate. Supports primary, secondary, tertiary, and semantic (error, warning, info, success) variants in three sizes with full keyboard and disabled state support.

Variants

Use the variant input to choose the visual style. primary is the main call-to-action; secondary is for supporting actions; tertiary is a ghost/text button. Semantic variants communicate intent.

Base variants

<usli-button label="Primary"   variant="primary"   />
<usli-button label="Secondary" variant="secondary" />
<usli-button label="Tertiary"  variant="tertiary"  />

Semantic variants

<usli-button label="Delete"   variant="error"   />
<usli-button label="Warning" variant="warning" />
<usli-button label="Info"    variant="info"    />
<usli-button label="Continue" variant="success" />

Sizes

Three sizes: small, medium (default), and large.

<usli-button label="Small"  [primary]="true" size="small"  />
<usli-button label="Medium" [primary]="true" size="medium" />
<usli-button label="Large"  [primary]="true" size="large"  />

Disabled state

Add [disabled]="true" to prevent interaction. Works on both variants.

<usli-button label="Primary"   [primary]="true"  [disabled]="true" />
<usli-button label="Secondary" [primary]="false" [disabled]="true" />

Content projection

Pass content as children instead of the label input for richer markup.

Save changes Cancel 
<usli-button [primary]="true">Save changes</usli-button>
<usli-button>Cancel</usli-button>

Invoker Commands New in browsers

The Invoker Commands API lets a button open, close, or toggle another element — no JavaScript required. Use commandFor to point at a target element's id, and command to specify what to do: toggle-popover, show-modal, or close. Available in Chrome 130+, Edge 130+, Firefox 135+, and Safari 18.4+.

Popover

Native Popover
No JavaScript — the browser handles show/hide via the Invoker Commands API. 
<usli-button label="Toggle Popover" variant="primary"
  commandFor="my-popover" command="toggle-popover" />

<div id="my-popover" popover class="invoker-popover">
  <div class="popover-header">Title</div>
  <div class="popover-body">Content</div>
</div>

Modal Dialog

Confirm action

This dialog was opened with command="show-modal" — zero JavaScript. 

<usli-button label="Open Dialog" variant="primary"
  commandFor="my-dialog" command="show-modal" />

<dialog id="my-dialog" class="invoker-dialog">
  <div class="modal-header">
    <h5 class="modal-title">Confirm action</h5>
  </div>
  <div class="modal-body">Dialog content</div>
  <div class="modal-footer">
    <usli-button label="Cancel" variant="secondary"
      commandFor="my-dialog" command="close" />
    <usli-button label="Confirm" variant="primary"
      commandFor="my-dialog" command="close" />
  </div>
</dialog>

API

Inputs

Name Type Default Description
primary boolean false Filled (primary) vs outlined (secondary) variant.
size 'small' | 'medium' | 'large' 'medium' Controls padding and font size.
label string 'Button' Text label. Overridden by projected content.
disabled boolean false Disables the button and blocks pointer events.
backgroundColor string | undefined undefined Optional inline background-color override.
commandFor string | undefined undefined Invoker Commands: id of the element this button controls (popover or dialog).
command string | undefined undefined Invoker Commands: action to invoke — toggle-popover, show-modal, or close.

Outputs

NameTypeDescription
clicked OutputEmitterRef<MouseEvent> Emits the native MouseEvent on click.
